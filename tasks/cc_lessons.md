# CC Lessons

- When a user corrects a geometric edge case, prefer extending the core math with a deterministic fallback and add regression tests for that edge case immediately. Example: parallel infinite bars should not raise in `geometry.py`; use the Appendix A parallel-normal construction for distance and return a canonical closest-point parameter pair instead of failing.
- When a user asks for FK visualization, do not draw independently solved endpoint frames if the intended behavior is a kinematic tree. Build the viewer as a true parent-child chain so upstream joint motion propagates to all downstream links, URDF-style.
- When a Rhino command uses core geometry helpers that already support an edge case, remove redundant script-level rejection logic and let the shared geometry path handle it. Example: `t1_bar_axis.py` should not reject parallel bars once `closest_params_infinite_lines` supports them.
- When a user specifies the exact package and version for a new dependency, add that pinned dependency to the repo manifest first and install from the manifest rather than doing an ad-hoc package install. Example: use `pybullet_planning==0.6.1` in `requirements.txt` for the PyBullet viewer workflow.
- When an all-solutions geometry solver visualizes overlapping branches, do not assume the sign pairs uniquely identify geometry. Keep all distinct converged minima across seeds, dedupe by geometric equivalence, and add a regression for any missing mirrored branch.
- When a pytest visualization mode pauses on `input()` or opens interactive viewers, make `--viz` disable capture automatically so `pytest ... --viz` behaves like `-s` without requiring a second flag.
- Rhino 8 Python 3 scripts must be launched through the ScriptEditor run command in toolbar macros and docs. Do not wire them to `_-RunPythonScript`, which uses the legacy parser and will fail on Python 3 syntax like function annotations.
- Rhino CPython interop should not assume Rhino point values are plain tuples. Convert `Point3d`-like objects explicitly via `.X/.Y/.Z` before passing them into NumPy, or the scripts will fail on selections and picked points.
- When a Rhino workflow starts from a user-selected bar that should be repositioned, preserve its original length and attributes by moving the existing object instead of deleting it and recreating a default-length replacement.
- When users want construction geometry preserved in Rhino, bake explicit reference objects at the moment the result is finalized. For shortest-contact segments, assign a distinct object color and set the object print width source to by-object so the weight is not inherited from the layer.
- When Rhino shows multiple candidate solutions for user selection, preview the supporting construction geometry for each candidate too, and match its color to the candidate axis so the grouping stays visually unambiguous.
